Discuss the scope of var, let, and const
  • var is the oldest keyword in JavaScript that is used to declare variables in JavaScript. Depending on where a var is declared, it can have two types of scope. Either global or functional scope.
  • let is another keyword used for declaring variables in JavaScript. let variable only has block scope which means it can only be accessed within the block it was declared..
  • const is Javascript keyword to declare a variable that remains constant which means you can't update the value of a variable when it's declared using const. Like let keyword, const also has block scope
Tell us the use cases of null and undefined
    Both null and undefined has different meaning when used in different contexts.
  • null means empty or unknown value.
    • Specially when a variable or object has no value i.e empty. One can use null to to assign empty value if needed so.
  • undefined means a missing value or a value that wasn't declared already. if a variable is declared without assigning a value then it gives undefined.
